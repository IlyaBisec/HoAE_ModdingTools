1. ЭФФЕКТЫ	
	Эффект имеет время анимации, на протяжении которого его частицы отображаются на экране.
Он состоит из одного или более дочерних эмиттеров.


2. ЭМИТТЕРЫ
В функции эмиттера входит рождение частиц в заданной точке пространства. 
Эмиттер имеет начальное время действия и продолжительность - в этот промежуток времени он рождает частицы.
Частицы, рожденные эмиттером могут жить на протяжении времени анимации родительского эффекта.
Узлы эмиттеров в редакторе обозначаются розовым цветом.

Общие параметры каждого эмиттера:
			- StartTime		- время, когда эмиттер начинает действовать (относительно начала анимации эффекта)
			- TotalTime		- время действия одного цикла эмиттера, в секундах
			- Looped		- зациклить действие эмиттера
			- WorldSpace	- частицы находятся в мировом пространстве. Если отключено, то все частицы локальны 
								относительно эмиттера
			- TimeToLive	- среднее значение времени жизни испускаемых частиц, сек
			- TimeToLiveV	- вариация времени жизни, сек
			
Количество частиц, которое рождается в каждый момент времени, определяется конкретным типом эмиттера:
			+ Constant	- частицы рождаются с постоянной интенсивностью
			+ Burst		- все частицы рождаются в начальный момент
			+ Ramp		- интенсивность рождения задается с помощью ключей на протяжении времени активности эмиттера 
			+ Static	- представляет собой единственный объект, живущий вечно
			+ Modular	- состоит из нескольких дочерних эмиттеров

3. ОПЕРАТОРЫ
Эмиттер всего лишь создает частицы, дальнейшие действия над ними выполняются с помощью так называемых операторов,
которые содержатся в эмиттере.
Операторы делятся на четыре типа:

-Initialize - те, которые задают начальные параметры родившейся частицы. Инициализаторы могут задавать:
			+ Начальное положение частицы относительно эмиттера (Placers: Sphere, Box, Cone, Line, 
																	Cylinder, TerrainRenderer, Mesh)
				Узлы Placers в редакторе обозначаются фиолетовым цветом.
			+ Начальную скорость (направление и величину) (Shooters: Direct, Radial, Cone, Ramp)
				Узлы Shooters в редакторе обозначаются голубым цветом.
			+ Начальные текстурные координаты (UVInit, FrameInit)
			+ Начальный цвет и размеры (ColorInit, SizeInit) 
			Обозначаются оранжевым цветом.

-Render		- отвечают за то, как частицы отображаются на экране:
		Здесь задаются общие параметры рендеринга частиц:
			+ Текстура 
			+ Аддитивный/мультипликативный рендеринг
			+ Интенсивность: обычная, усиленная, безумная
			+ Использование z-буффера (на предмет чтения и записи)
			+ Z-bias - Частицы кластера будут рендериться немного "ближе" в экранном пространстве, чем остальная сцена, что может 
				помочь уменьшить эффект линейного пересечения частиц с геометрией сцены. Может принимать значения [0..3]
			
		Конкретные типы рендереров могут отображать частицы как:
			+ Затекстуренные прямоугольники (Billboard)
			+ Соединенные биллборды (Chain)
			+ Геометрические примитивы (Sphere, Cone)
			+ Целые модели (ModelObject)
			+ Накладки на ландшафте (TerrainDecal) и поверхности воды (WaterDecal)
			
		Узлы визуализации в редакторе обозначаются зеленоватым цветом.
	
-Affect		- изменяют параметры частицы на протяжении их жизни
		Изменяться могут:
			+ Текстурные координаты (FrameAnimate, UVAnimate)
			+ Скорость (Force, Drag, Wind, Turbulence, Vortex, Follow)
			+ Прозрачность (AlphaFade, AlphaRamp)
			+ Цвет (ColorFade, ColorRamp)
			+ Размеры (SizeFade, SizeRamp)
		Узлы аффекторов в редакторе обозначаются желтым цветом.
			
-Trigger	- отслеживают состояние частицы с тем, чтобы произвести некоторое действие по событию:
			+ Рождение частицы (Birth)
			+ Прошествие промежутка времени (Timer)
			+ Столкновение частицы с чем-то (HitPlane, HitGround, HitWater)
			+ Смерть частицы (Death)
		Пример результата срабатывания триггера - это рождение экземпляра другого эффекта в 
	текущем положении рассматриваемой частицы. 
		Действия, которые производятся с частицей в результате срабатывания триггера, могут быть такими:
			+CreateEffect	- создать на месте частицы экземляр другого эмиттера, который нужно 
								подсоединить как дочерний узел для триггера
			+Death			- убить частицу
			+Deflect		- отразить частицу, если это имеет смысл для данного триггера
			+ResetAge		- сбросить возраст частицы в ноль
			+Stop			- остановить частицу
			+StopRotation	- остановить вращение частицы
		
		Узлы аффекторов в редакторе обозначаются голубым цветом.
		
-Target		- используют некую целевую точку для проведения действий над частицами
			+ Эффект молнии (Lightning)
			+ Притягивание частиц к цели (Homing)
		Узлы Target в редакторе ядовито-желтые
		
	Красным цветом обозначаются те узлы, которые пока еще не работают как положено.
